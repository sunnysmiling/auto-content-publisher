# Auto Content Publisher - 需求规格 (requirements.md)

## 1. 项目概述

**项目名称**: Auto Content Publisher  
**项目类型**: 自动化内容采集与发布系统  
**核心功能**: 全自动从指定来源采集文章数据，AI 二次加工后自动发布到公众号  
**目标用户**: 内容创作者、自媒体运营者

---

## 2. 功能需求

### 2.1 核心功能模块

| 模块 | 功能描述 | 优先级 |
|------|----------|--------|
| **数据采集** | 从指定 RSS 源 / 目标网站自动抓取文章 | P0 |
| **内容处理** | AI 改写、去重、摘要生成、伪原创 | P0 |
| **发布渠道** | 公众号自动发布（需公众号开放平台 API） | P0 |
| **定时任务** | 定时采集、定时发布 | P1 |
| **管理后台** | 查看已发布/待发布内容、人工审核 | P2 |

### 2.2 数据采集

- 支持 RSS 订阅源抓取
- 支持 HTTP 请求抓取网页内容
- 支持多个来源并行采集
- 自动过滤无效/重复内容
- 增量更新，只抓取新内容

### 2.3 内容处理

- AI 改写（保留核心意思，更换表达方式）
- 敏感词过滤
- 标题生成/优化
- 自动配图（从原文提取或 AI 生成）
- 内容摘要提取

### 2.4 发布

- 公众号草稿箱自动创建
- 支持图文消息发布
- 支持草稿箱保存后人工确认发布
- 发布结果状态回执

### 2.5 定时任务

- 采集频率：可配置（默认每 2 小时）
- 发布频率：可配置（默认每天 8:00 / 12:00 / 20:00）

---

## 3. 技术架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  采集层      │ ──▶ │  处理层      │ ──▶ │  发布层      │
│ (Scraper)   │     │ (AI Process)│     │ (Publisher) │
└─────────────┘     └─────────────┘     └─────────────┘
       │                   │                   │
       └───────────────────┴───────────────────┘
                           │
                    ┌──────┴──────┐
                    │  调度中心    │
                    │  (Scheduler)│
                    └─────────────┘
```

### 技术栈

- **语言**: Python 3.10+
- **采集**: requests, feedparser, BeautifulSoup4, Selenium (可选)
- **AI 处理**: OpenAI API (GPT-4) 或兼容 API
- **存储**: SQLite (轻量) 或 MySQL
- **定时**: APScheduler
- **公众号**: 微信公众号草稿箱 API

---

## 4. 数据模型

### 4.1 Article (文章)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| source_url | TEXT | 原始链接 |
| original_title | TEXT | 原文标题 |
| original_content | TEXT | 原文内容 |
| processed_title | TEXT | 加工后标题 |
| processed_content | TEXT | 加工后内容 |
| summary | TEXT | 摘要 |
| status | TEXT | pending/processing/published/failed |
| created_at | DATETIME | 创建时间 |
| published_at | DATETIME | 发布时间 |

### 4.2 Source (数据源)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| name | TEXT | 源名称 |
| url | TEXT | 源 URL |
| type | TEXT | rss / http |
| enabled | BOOLEAN | 是否启用 |
| crawl_interval | INTEGER | 采集间隔(分钟) |

---

## 5. 非功能需求

- **稳定性**: 7×24 小时运行，异常自动重试
- **可配置**: 所有参数可通过配置文件调整
- **可扩展**: 方便添加新数据源和新发布渠道
- **日志**: 完整运行日志，便于排查问题
